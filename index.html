Shoplive 어드민 워크스루

실제 Shoplive 어드민 화면 위에 단계별 안내를 띄워, 고객사가 직접 클릭하며 방송을 만들어 보게 하는 도구입니다. 설치가 필요 없습니다. 아래 링크를 북마크바에 드래그해서 저장하고, 어드민 화면에서 누르면 됩니다.

1. 실행할 플로우 선택

선택한 플로우가 아래 "워크스루 실행" 북마클릿에 반영됩니다.

2. 북마클릿 저장 (드래그)

워크스루 실행 플로우 빌더

위 버튼을 브라우저 북마크바로 드래그하세요. (클릭하면 이 페이지에서는 동작하지 않습니다 — 실제 어드민 화면에서 눌러야 합니다.)

[임시] 커밋 고정 북마클릿

jsDelivr가 @main 최신 커밋을 아직 못 따라와서, 지금 이 커밋()에 고정된 임시 북마클릿입니다. @main이 정상화되면 위 "2. 북마클릿 저장"의 링크를 다시 쓰시고 이 섹션은 지워도 됩니다.

워크스루 실행 (고정) 플로우 빌더 (고정)

engine.js / 플로우 JSON을 고친 뒤에는

GitHub에 push해도 jsDelivr가 예전 파일을 한동안 그대로 서빙할 수 있습니다(쿼리스트링 버전 표시는 참고용일 뿐 실제 캐시 무효화 효과는 없습니다). 이 버튼을 누르면 지금 선택된 플로우 기준으로 engine.js와 플로우 JSON의 jsDelivr 캐시를 강제로 갱신합니다.

3. 사용 순서

  1. 이 레포(jameslee-lang/admin-test)에 push 되어 있어야 북마클릿이 정상 동작합니다.
  2. GitHub Pages를 이 폴더로 켜거나, 그냥 이 index.html을 로컬에서 열어도 됩니다.
  3. 위 두 링크(워크스루 실행, 플로우 빌더)를 북마크바에 드래그해 저장합니다.
  4. 실제 Shoplive 어드민 페이지로 이동한 뒤 워크스루 실행 북마클릿을 누릅니다.

새 플로우를 만들거나 고칠 때

JSON이나 CSS 선택자를 몰라도 됩니다. 실제 어드민에서 플로우 빌더 북마클릿을 누르면 화면에 패널이 뜨고, 순서대로 요소를 클릭 → 그 자리에서 제목/설명/자동 진행 조건을 입력 → "이 스텝 추가"만 반복하면 됩니다. 다 만들었으면 "완성된 JSON 만들기"를 눌러 전체 복사하거나 파일로 다운로드해서, 그 내용을 그대로 flows/<id>.json에 붙여넣으면 끝입니다. 자세한 방법은 README.md를 참고하세요.
