# 방송 등록 워크스루 — 내용 검증

`broadcast-register.json`에 들어간 단계가 실제 방송 등록 흐름과 맞는지 여기서 검토·수정해주세요.
이 파일은 사람이 읽고 고치는 용도이고, JSON은 여기 내용 + 셀렉터를 합친 실행용 파일입니다.
이 파일을 고친 뒤 "review.md 반영해줘"라고 하면 JSON에 동기화합니다. 셀렉터까지 아시면 JSON을 바로 고치셔도 됩니다.

## 기준 문서
- `../../admin-training-checklist.md` 1단계(방송 만들기) / 2단계(팔 상품 넣기) — 이미 검증된 내용
- `../../admin-training-script.md` 1단계 — 같은 내용의 구어체 버전

## 현재 구현된 단계 (JSON 순서 그대로)

1. **[정보 안내]** 방송 등록 시작 — 화면 요소 없이 안내 문구만
2. **[확인 필요]** 방송 등록 메뉴로 이동 — 셀렉터 없음 (목록 화면을 아직 안 열어봐서 못 뽑음). "방송 추가" 버튼 클릭 → 편집 화면(`/campaigns2/...`)으로 이동한다고 가정 중. 실제로 그런지, URL 패턴이 맞는지 확인 필요.
3. 방송 제목 입력 (`#campaign-form_title`) — 체크리스트 1단계와 일치
4. 방송 시작 일시 설정 (`#campaign-form_scheduledAt`의 날짜 입력란) — 체크리스트 1단계와 일치. **시각 입력란은 자동 감지 안 되고 안내 문구로만 처리 중** (날짜만 입력해도 다음 단계로 넘어감)
5. 상품 탭으로 이동 (`#rc-tabs-0-tab-goods`) — 체크리스트 2단계 진입점. 화면 폭에 따라 "···" 더보기 메뉴 안에 숨어 있을 수 있음. **고객사 권한 계정에서는 탭 구성 자체가 다를 수 있어 미확인**
6. 상품 연결 (`button.add-group.group`) — 체크리스트 2단계 "방송에 추가"에 해당. **셀렉터 안정성 낮음** — styled-components 해시 클래스라 배포되면서 바뀔 수 있음
7. **[확인 필요]** 등록 완료 (`#campaign-form .save-button-wrap button`, "저장" 버튼) — 이 버튼 하나로 제목·시간·상품이 한 번에 저장되는지, 아니면 섹션마다 따로 저장해야 하는지 미확인. 체크리스트에는 이 저장 동작 자체가 명시돼 있지 않음.

## 빠진 것 / 결정 필요
- [ ] 체크리스트 4단계(금칙어)는 "방송 시작 전 필수"라고 되어 있는데, 이 "기본 흐름"에 포함시킬지 별도 플로우(`banned-words.json`?)로 뺄지
- [ ] 체크리스트 5단계(공개) 포함 여부
- [ ] 상품 자체를 등록하는 과정(상품 데이터 생성)은 이 플로우에 없음 — "이미 등록된 상품이 있다"고 가정 중. 처음 쓰는 고객사는 상품이 아예 없을 수도 있는데, 그 경우 안내가 필요한지
- [ ] 상품 그룹(합구매) 관련 스텝은 아예 없음 — 넣을지 말지

## 수정 방법
1. 순서·문구·포함 여부는 이 파일에서 체크리스트처럼 편하게 고치기
2. 셀렉터가 필요한 새 단계는: `index.html`에서 "요소 선택기" 북마클릿으로 실제 요소 클릭 → 나온 셀렉터를 이 파일에 메모
3. 다 고쳤으면 저한테 "review.md 반영해줘"라고 알려주기 → JSON 동기화
