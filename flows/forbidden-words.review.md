# 방송 금칙어 등록(자세히) 워크스루 — 내용 검증

`forbidden-words.json`에 들어간 단계가 실제 금칙어 등록 흐름과 맞는지 여기서 검토·수정해주세요.
이 파일은 사람이 읽고 고치는 용도이고, JSON은 여기 내용 + 셀렉터를 합친 실행용 파일입니다.
이 파일을 고친 뒤 "review.md 반영해줘"라고 하면 JSON에 동기화합니다. 셀렉터까지 아시면 JSON을 바로 고치셔도 됩니다.

## 기준 문서
- `../../admin-training-checklist.md` 4단계(금칙어, 방송 시작 전 필수)

## 이 플로우를 만든 방식
Claude in Chrome으로 실제 어드민(`adm.shoplive.cloud`)에 로그인된 "어드민 교육" 테스트 캠페인 화면을
직접 읽어서 만들었습니다. 화면 이동·클릭은 James님이 직접 하고, Claude는 클릭하지 않고 화면만 읽어서
셀렉터를 뽑고 제목·설명을 붙였습니다 (플로우 빌더 북마클릿의 "패널에 타이핑하는 부분"을 대신한 방식).
스텝 5(추가하기)는 실제로 눌러서 "테스트금칙어"라는 값이 정상 등록되는 것까지 확인했습니다.

## 현재 구현된 단계 (JSON 순서 그대로)

1. 방송 금칙어 관리 (`#campaign-form_campaignBannedWordCount > button`) — "관리" 버튼 클릭
2. 금칙어 추가 (`.ant-modal-body .title-wrapper button`) — 팝업 안의 "금칙어 추가" 버튼. **셀렉터를 직접 골랐음**: 이 페이지에 antd 모달이 여러 개(자동 추첨, 테마 프리셋 등) 숨겨져 있어서 자동 생성 셀렉터(`div:nth-of-type(33)...`, 모달 마운트 순서 의존)보다 안정적인 클래스 조합으로 대체
3. 금칙어 입력 (`#rc_select_0`) — **셀렉터 안정성 낮음**: rc-select 라이브러리가 마운트 순서대로 붙이는 자동 카운터 ID라, 이 페이지에 다른 select 컴포넌트가 먼저 마운트되면 번호가 바뀔 수 있음
4. **[확인 필요]** 매칭 범위 선택 (`#form-banned-word_matchStrategyType`, "부분/단어/문장" 라디오) — 설명 문구가 실제 동작 확인 없이 UI 문구만 보고 추측한 내용입니다. 정확한 매칭 규칙으로 고쳐주세요.
5. 추가하기 (`#campaign-form > div:nth-of-type(4) > ...`) — 실제로 눌러서 "테스트금칙어"가 등록되는 것까지 확인함. **셀렉터가 김** (nth-of-type 경로가 길게 이어짐, `#campaign-form`에 고정되어 있긴 하지만 폼 구조가 바뀌면 깨질 수 있음)

## 빠진 것 / 결정 필요

- [ ] 매칭 범위(부분/단어/문장) 설명 문구 — 실제 동작 기준으로 검증 필요 (위 4번 항목)
